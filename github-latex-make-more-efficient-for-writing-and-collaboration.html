What is Git and what are the benefits of Git?

Git is a version control system primarily intended for use with program code. In short, Git is a tool for collaborating on code that makes it possible to handle changes and history.

Some benefits of Git [1]

  • Keep track of all files in a project
  • Record any changes to project files
  • Restore previous versions of files
  • Compare and analyze code
  • Merge code from different computers and different team members.

4 The branch

If you know how to use branch cleverly, this will greatly improve your work efficiency. When you create a local repository, master branch is the default branch or main branch. The advantage of branches is that each branch is independent during the development process. If during the development process of one branch fails, it will not have any impact on other branches and main branch. Just delete the failed branch and start again. As the owner of the branch, you can merge the two branch at any time. Of course, the most important thing is that branch can effectively avoid unnecessary conflicts.

In the picture we see the master branch and two branches, let call them feature and feature . Imagine you want to develop a new feature. Then you can divide this feature into two small features for independent development, which are feature and feature. When you are finish with one of them, then you can merge them into the master branch. Simply, branch can effectively help you divide big problem into different small problems and then merge them into the main branch. If one of the feature is failed to develop, it will not affect the entire project. At the same time, you can imagine that if there is no branches, then you have to develop all new features in the main branch. If there is a mistake in the development process, the final result may be irreversible.

5 Overwrtting with merge

Earlier we introduced branches and how to create branches. Then they can be overwritten between different branches. By default, the merge command merges different branches. If you want to write something in one branch and then merge to another branch, you can use this command

Unfortunately, not all mergers can be successful. Conflicts will appear in many merge situations. This is because you make different changes to the same part of the same file in two different branches, Git cannot merge them cleanly, and conflicts will occur. If conflicts appear, the only solution is that you need to modify these files manually. After the modification, you need to execute the following commands to mark them as modifie resolved successfully git add <filename>.

What is Githut and waht is difference between GitHub and Git?

Git is the bow, your code is the arrow, and Github is the target.

Simply, git is a software, it is also a local “warehouse” at your pc where you can save different versions of the code and documents. GitHub is an online “warehouse”, where you can also save your code and its version.

Create a repository at GitHub

We already know how to create a lokalrepository, that is use the command git init. All you need to do now is to create an online repository. The advantage of online repository is that can be shared and edited by multiple people.

In Git you can create two different types of repositories, public and private. Public is equivalent to open source, so everyone can download your repository. If you want to use GitHub for group work or study, it is best to use a private repository. In this repository, only invited people can edit and view.

Keep in mind that rules about cheating and plagiarism may make it necessary to use private repositories for group work!
